This example will demonstrate how to properly assemble all of the required HTML to display a form consisting of the three basic elements of an input, select and textarea, along with a submit button.

If you're familiar with how Bootstrap asembles its forms, this example should be very easy. We optionally use the <form> tag which may not necessary if you're using a JavaScript library or framework in your application but is consistent with properly formatted HTML. We also use the <fieldset> tag in order facilitate the use of assistive technology. Neither tag is associated with Magma styling and will notbe visible in the display.

Secondly, we wrap our form elements in a div or other similar tag using the .cenform-group class, which is very similar to the .form-group class you would see in Bootstrap. This will apply all of the neccessary styles to its child elements and can also be used to asign validation and visibility states.

Finally, we apply the .cenform-control class to our form elements in order to style them according to the Magma style guide. Again, this is very similar to the .form-control class you'll find in Bootstrap.

The .cenform-control class should be added to all input, select and textarea form elements to syle them properly. The label tag does not need the .cenform-control class as it inherits its styles from the parent .cenform-group element and appear hidden by default.

Our submit buttons can be configured in a variety of ways, with the default being represented by a simple button, right aligned, directly after the last form element. To achieve this result, we create a .cenform-submit element, designate the alignment we want with the .align-right class and insert the button markup with the appropriate classes, in this case .btn and .btn-success.
